# **Процесс нормализации БД**

## **1. Исходная таблица (ДО нормализации)**

```sql
ВсеДанные (
    id,
    задача,
    время_начала,
    время_окончания,
    имя_пользователя,
    email_пользователя,
    название_проекта,
    теги
)
```

## **2. Проблемы**

### **Избыточность**
- Данные пользователя повторяются

### **Аномалии**
- Изменить email = обновить все записи

### **Неатомарность**
- Теги в одной ячейке

## **3. Нормализация до 1NF**

### **Убраны неатомарные значения:**
- Теги → отдельная таблица **Tags**
- Связь многие-ко-многим → **Session_Tags**

## **4. Нормализация до 2NF**

### **Устранены частичные зависимости:**
- Данные пользователя → таблица **Users**
- Данные проекта → таблица **Projects**
- В **Time_Sessions** остались только FK

## **5. Нормализация до 3NF**

### **Устранены транзитивные зависимости:**
- Настройки → **User_Settings**
- Отчеты → **Reports**

## **6. Итог (7 таблиц)**

- **Users** – данные пользователя  
- **Projects** – проекты  
- **Time_Sessions** – сессии времени  
- **Tags** – теги  
- **Session_Tags** – связь тегов и сессий  
- **Reports** – отчеты  
- **User_Settings** – настройки  

**Каждая таблица зависит только от своего PK.**
