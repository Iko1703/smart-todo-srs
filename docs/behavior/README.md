# Диаграммы поведения системы

Документация отражает поведение приложения **«Трекер времени»**, включая сценарии использования (Use Case) и основные последовательности взаимодействия (Sequence Diagrams).

---

## Use Case Диаграмма

Описание основных сценариев использования приложения **«Трекер времени»**.

### Акторы:
- **Пользователь** — основной пользователь системы, запускает таймер, создаёт записи, работает со статистикой.
- **Администратор** *(опционально)* — управляет системными настройками и резервными копиями.

### Основные Use Cases:
- **Запуск и остановка таймера**  
  Используется для отслеживания длительности активности (рабочей сессии).
- **Создание записи вручную**  
  Пользователь может указать время начала и окончания без таймера.
- **Просмотр, редактирование и удаление записей**  
  Управление историей времени.
- **Фильтрация, поиск и сортировка записей**  
  Фильтр по дате, проекту, категории; поиск по описанию.
- **Работа с проектами и категориями**  
  Создание, редактирование и привязка к записям.
- **Просмотр статистики и диаграмм**  
  Дневные, недельные, месячные отчёты; распределение по категориям.
- **Экспорт отчётов (CSV/JSON)**  
  Выгрузка данных за период.
- **Настройки пользователя**  
  Язык, тема интерфейса, уведомления о длительных сессиях.

---

## Диаграммы последовательности

### 1. Запуск и остановка таймера
Показывает процесс старта и завершения временной сессии:
- создание новой активной сессии;
- назначение уведомления о длительной работе;
- перевод сессии в запись (record) после остановки;
- обновление UI.

### 2. Ручное добавление записи
Демонстрирует процесс создания записи без таймера:
- ввод start/end времени;
- валидация диапазона;
- сохранение в базу;
- отображение в списке записей.

### 3. Экспорт отчёта
Отображает механизм экспорта данных:
- получение записей за интервал;
- формирование CSV/JSON;
- предоставление ссылки на скачивание.

### 4. Комплексный бизнес-процесс (ежедневный workflow)
Полный сценарий работы пользователя:
- вход в приложение и загрузка состояния;
- запуск/пауза/резюмирование таймера;
- уведомления о длительной активности;
- остановка таймера → создание записи;
- ручные записи при необходимости;
- автоматическая синхронизация;
- экспорт отчёта или создание резервной копии.
